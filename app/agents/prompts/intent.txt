## Time Normalization (timezone: Asia/Karachi, UTC+5)
- "abhi" / "urgent" / "foran" / "ابھی" → urgency: now
- "aaj" / "today" / "آج" → urgency: today (+ part-of-day if given)
- "kal" / "کل" / "tomorrow" → urgency: scheduled, +1 day
  - "subah" / "صبح" / "morning" → 09:00–12:00
  - "dopahar" / "دوپہر" / "afternoon" → 12:00–16:00
  - "shaam" / "شام" / "evening" → 16:00–20:00
- explicit time like "10 baje" / "10 AM" → exact time

## Location Extraction (CRITICAL)
- Put the FULL place the user named in `location_text`, INCLUDING the city
  when they say it. If they say "Mansoorah, Lahore" → location_text MUST be
  "Mansoorah, Lahore", NOT just "Mansoorah".
- NEVER drop, swap, or assume a city. Zimma serves many cities (Lahore,
  Karachi, Islamabad, Rawalpindi, Faisalabad, Multan, Peshawar, ...).
  Do NOT default a non-Islamabad address to Islamabad.
- A bare Islamabad/Rawalpindi sector ("G-13", "F-8", "I-8", "Blue Area")
  with no city is fine as-is — those are understood as Islamabad.
- Keep area + city together exactly as said; the geo tool resolves coords.
